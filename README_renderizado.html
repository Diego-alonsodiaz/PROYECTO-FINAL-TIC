🧪 Evaluación Comparativa de Rendimiento: Máquinas Virtuales vs Contenedores Docker

Este proyecto evalúa el rendimiento de contenedores Docker frente a máquinas virtuales VirtualBox, utilizando un conjunto de pruebas controladas y análisis comparativo detallado.

📘 ¿Qué son las Máquinas Virtuales y los Contenedores?

🖥️ Máquinas Virtuales (VM)

Una VM simula un hardware físico completo, permitiendo ejecutar un sistema operativo completo sobre un hipervisor. Proporcionan alto aislamiento, pero con mayor sobrecarga de recursos.

📦 Contenedores (Docker)

Los contenedores ejecutan procesos aislados que comparten el kernel del sistema operativo. Son más livianos, rápidos de iniciar y escalan eficientemente, aunque con menor aislamiento comparado a las VMs.

🧪 Entorno de Pruebas

Las pruebas se han ejecutado bajo condiciones controladas, con escenarios replicables tanto en Docker como en VirtualBox. Las tareas consisten en la ejecución de código en C compilado, con uso intensivo de CPU, utilizando distintos niveles de paralelismo: 1, 2, 4, 6 y 8 hilos.

Cada prueba mide:

  • ⏱️ Tiempo total de ejecución
  • 🧠 Uso de RAM
  • 🔥 Uso de CPU por segundo
  • 🚀 Tiempo de arranque del entorno

Los datos se recogen automáticamente y se almacenan en formato .csv para facilitar el análisis posterior.

📈 Análisis de Resultados

🔥 Uso de CPU durante el Benchmark

  • Se observa un pico inicial de uso de CPU en todos los casos, seguido de estabilización.
  • A mayor número de hilos, la carga tiende a mantenerse más constante, pero también se presentan más picos intermedios.
  • El rendimiento se estabiliza cerca del 6% de uso medio, aunque con más variabilidad a 6 y 8 hilos.

🧠 Uso de Memoria durante el Benchmark

  • La memoria usada se mantiene en torno a los 8500 MB en todos los escenarios.
  • El uso de memoria es ligeramente mayor en configuraciones con más hilos.
  • Docker presenta mayor estabilidad en consumo de memoria en comparación con la VM, que tiene más fluctuaciones a partir de 6 hilos.

📊 Comparativa y Visualización

Los notebooks comparar_resultados.ipynb y vm_vs_docker.ipynb incluyen:

  • Gráficos de comparación entre Docker y VirtualBox
  • Evaluación visual de escalabilidad al aumentar el número de hilos
  • Análisis de eficiencia multithread
  • Conclusiones respaldadas por datos empíricos

✅ Conclusiones

Criterio Docker VirtualBox
Tiempo de arranque ✅ Rápido (~1s) ❌ Lento (~30s)
Consumo de recursos ✅ Bajo ❌ Alto
Aislamiento ❌ Parcial ✅ Completo
Portabilidad ✅ Muy alta ⚠️ Moderada
Rendimiento multihilo ✅ Escalable ⚠️ Menor eficiencia
  • Docker es ideal para despliegue rápido y pruebas ligeras.
  • VirtualBox sigue siendo útil donde se requiere aislamiento completo.
  • A mayor número de hilos, Docker mantiene una mejor eficiencia en CPU y memoria.
